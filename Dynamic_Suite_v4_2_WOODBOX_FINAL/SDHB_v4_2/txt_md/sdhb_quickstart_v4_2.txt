SDHB (Suite Dynamic Home Bus) — QUICKSTART (Suite v4.2 · BUS v4.2)
===================================================================

1) Instalar BUS/SDHB
--------------------
Copia a /config/packages/ (3 archivos):
- sdhb_bus_helpers_v4_2.yaml
- suite_dynamic_home_bus_v4_2.yaml
- sdhb_bus_automations_v4_2.yaml
Reinicia Home Assistant.

2) Verificación rápida
----------------------
Debe existir:
- input_boolean.sdhb_enabled
- script.sdhb_publish_intent
- script.sdhb_heartbeat
- script.sdhb_feedback

Para activar coordinación:
- input_boolean.sdhb_enabled = ON

3) Prueba manual (Developer Tools → Services)
---------------------------------------------
Llama a script.sdhb_publish_intent con:
- source: ds_w1
- target: dc_zone01
- intent: request_solar_shield
- priority: 20
- ttl_s: 900
- payload: JSON string

Luego verifica:
- sensor.sdhb_winner_dc_zone01 cambia a request_solar_shield (mientras dure TTL).

Extras:
- script.sdhb_clear_intent (limpieza explícita por source)

Optional: include sdhb_testing_integration_optional_v4_2.yaml to run SDHB integration tests (disabled by default).


Optional tests:
- E2E (DC→SDHB→DS): include `sdhb/sdhb_testing_e2e_optional_v4_2.yaml` and enable `input_boolean.sdhb_e2e_tests_enabled`.
  Select scenario in `input_select.sdhb_e2e_test_scenario`.

Docs:
- SDHB Contracts: txt_md/sdhb_contracts_v4_2.md

Optional tests:
- sdhb_testing_recovery_optional_v4_2.yaml (TTL expiry & fallback)


Roadmap extra aplicado:
- sdhb_ttl_cleanup_5m: limpia slots expirados cada 5 min (cosmético + claridad de helpers)
- sdhb_startup_feedback_guard: limpia feedback_applied sucios al arrancar
- sensor.sdhb_intent_history: historial corto (~15) de cambios de winner (via evento sdhb_winner_changed)
